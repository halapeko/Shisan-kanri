# Shisan-kanri
資産管理用ツール

## 構成

- `index.html` — 資産定点観測帳（全資産の月次記録）＋ ポートフォリオ運用（AI管理）画面
- `data/portfolio.json` — 証券ポートフォリオの中枢データ（方針・基本配分・商品・保有・価格・履歴・アラート・AIログ）
- `scripts/update_portfolio.mjs` — 価格自動取得＋アラート生成スクリプト
- `.github/workflows/update-portfolio.yml` — 平日朝6:30(JST)に上記を自動実行し、結果をcommit

## ポートフォリオ運用（AI管理）の仕組み

目標：証券資産を**年率5%**で運用する。投資方針の原本は Google スプレッドシートの「投資ルール」タブ。

### 基本配分（期待リターン 年約5.0%）

| クラス | 配分 | 商品 | 口座 |
|---|---|---|---|
| 全世界株式 | 35% | eMAXIS Slim 全世界株式（オルカン） | NISAつみたて枠 |
| 米国株式 | 10% | eMAXIS Slim 米国株式（S&P500） | NISAつみたて/成長枠 |
| 先進国債券 | 15% | eMAXIS Slim 先進国債券インデックス | NISA成長枠 |
| 国内債券 | 5% | eMAXIS Slim 国内債券インデックス | NISA成長枠 |
| 金 | 10% | SBI・iシェアーズ・ゴールドファンド（ヘッジなし） | NISA成長枠 |
| 日本株式（高配当株・個別株） | 25% | Tracers 日経平均高配当株50＋個別株 | NISA成長枠→特定 |

- 投資信託 75%（方針の「7割以上」を充足。NISA枠を使い切ってから低コスト商品を特定口座で）
- 個別株は **30%以下** を上限としてアラート監視（判定は商品種別ベース。Tracers等の日本株投信は投信側にカウント）
- 個人向け国債は額面評価（valueOverride）、金現物は国際スポット×為替から円/gを自動算出
- 株式と低相関の債券・金を組み込み、下振れ時の打撃を総資産で抑える（投資ルール「分散」）

### 自動運用サイクル

1. **平日朝**: GitHub Actions が基準価額・株価を取得 → 評価額と配分を計算 → `data/portfolio.json` に記録
2. **アラート自動生成**: 基本配分から±3%超のドリフト／個別株30%超／半年ごと（1月・7月）の定期点検
3. **週次**: Claude が運用状況をレビューし、状況を報告
4. **画面**: GitHub Pages の `index.html` がJSONを読んで現在の運用状況・アラート・購入プランを表示

### 品質担保体制（週次レビューの工程分離）

課題・改善点を面で捉えて防ぐため、週次AIレビューは工程ごとに別担当（サブエージェント）が実施し、後工程が前工程の成果物を横断チェックする。

| 工程 | 担当 | 役割 |
|---|---|---|
| 1 | データ検証担当 | 価格取得の稼働・鮮度、評価額再計算による整合性検証。信頼できる数値サマリーを作成 |
| 2 | 運用分析担当 | ドリフト・投資ルール適合・実績リターンvs目標5%・NISA枠の分析と推奨アクション立案 |
| 3 | 品質・改善担当 | 工程1・2の横断レビュー（計算誤り・見落とし・ルール不整合の指摘）とシステム/運用の改善提案 |
| 4 | 統括 | 工程間の矛盾を裁定し結論を確定。修正・記録（aiLog/alerts）・ユーザー報告 |

### 使い方（手作業はこれだけ）

- **買った/売ったら**: Claude のチャットに「オルカンを◯口（◯円）買った」と伝える → JSONに記録され、以後は全自動
- **リバランス**: アラートが出たら、表示された金額を目安に毎月の買付配分を調整（原則売却しない）
- 価格・履歴・損益・アラートの更新に手作業は不要
